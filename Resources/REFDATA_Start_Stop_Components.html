Create MKV Init

[Arguments]

${Directory}

@{ContentsOfFile}

[Documentation]

Keyword *Create MKV Init* creates the mkv.init for any component. \n
*Parameter*:\n
*<Directory>* - Mandatory – Define the name of the file either using the complete path or only the name of the file. \n
*<ContentsOfFile>*: List variable containing the contents of the file.\n
_*The elements of list variable should be in the following format*_:\n
set {MKVCOMPONENT} \\t {REFDATA} \\r set {MKVUSER} \\t {sec_dev} \\r and so on. \n
*Example*: \n
Create MKV Init mkv.init @{ContentsOfFile} \n
If Only the name of the file is given then the file with the specified name is created in the working directory of the Robot Framework.\n
if full path is given along with the name of the file then file is created with in the given path. _*In Case of Windows Operating System, the path separator should be defined as “\\\\” and In case of Linux “/”*_. \n
The list variable should be defined in the set of elements as: set {MKVCSHOST} \\t {192.168.44.165} \\r

Create Directory And Copy Executable

[Arguments]

${ComponentExecName}

${DirectoryName}

${DirectoryPath}=

[Documentation]

*Create Directory And Copy Executable* keyword is the private keyword used by other keywords in this file. This keyword creates the directory with the name _*<DirectoryName>*_ under the path _*<DirectoryPath>*_ and also copies the _*Component Executable*_ in the created directory.\n
This keyword also returns the full path of the Executable directory i.e. path containing the Directory Path and the DirectoryName.\n
*Parameters*:\n
<ComponentExecName> – _*Mandatory*_ – Name of the executable to be copied in the execution directory. *The executable should be available in the Execution Directory of Robot Framework.*\n
<DirectoryName> – _*Mandatory*_ – Name of the directory to be created. \n
<DirectoryPath> - Optional – Define the full path under which directory with name <DirectoryName> would be created. _*In Case of Windows Operating System, the path separator should be defined as “\\\\” and In case of Linux “/”*_. \n
If the parameter <DirectoryPath> is not defined or left blank, then the directory with name <DirectoryName> will be created in the Execution Directory of Robot Framework.\n
*Example*: \n
Create Directory And Copy Executable REFDATA mkvrefdata638d2.exe E:\\\\RobotFrameworkExecutions\\\\IONTester \n
${fullPath} = E:\\RobotFrameworkExecutions\\IONTester\\REFDATA
