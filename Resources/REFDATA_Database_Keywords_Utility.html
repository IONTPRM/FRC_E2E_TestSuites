Get WhereClauseCondition

[Arguments]

${FieldNameList}

${conditionList}

${FieldValueList}

${LogicalOperatorList}










[Documentation]

This keyword is used to get the where clause condition required in the "Delete" & "Update" commands of SQL.\n

*Parameters:*\n <FieldNameList>: List of fields name in the database. The list element should be passed as the parameter but using "$" sign.\n <conditionList>: List of corresponding conditions for each field which will be evaluated against the value. The list element should be passed as the parameter but using "$" sign.\n

E.g. Possible Values could be "=", "!=", ">", ">=", "<", "<=" and so on based up on the type of DBMS.\n

<FieldValueList>: List of corresponding value for each field which will be evaluated against the condition. The list element should be passed as the parameter but using "$" sign.\n <LogicalOperatorList>: List of logical operators to be used in between multiple conditions.












${whereClauseCondition}=

Set Variable













${tempCondition}=

Set Variable













@{fieldList}=

Convert To List

${FieldNameList}












@{conditionList}=

Convert To List

${conditionList}












@{valueList}=

Convert To List

${FieldValueList}












@{operatorList}=

Convert To List

${LogicalOperatorList}












${noOfElementsFieldName}=

Get Length

${fieldList}












${noOfElementsCondition}=

Get Length

${conditionList}












${noOfElementsFieldValue}=

Get Length

${valueList}












${noOfElementesLogicalOperator}=

Get Length

${operatorList}












${maxIndexLogicalOpeator}=

Evaluate

${noOfElementesLogicalOperator}-1












Should Be True

'${noOfElementsFieldName}'=='${noOfElementsCondition}' and '${noOfElementsCondition}'=='${noOfElementsFieldValue}'













:FOR

${n}

IN RANGE

0

${noOfElementsFieldName}











${tempCondition}=

Catenate

${whereClauseCondition}

@{fieldList}[${n}]

@{conditionList}[${n}]

@{valueList}[${n}]









${whereClauseCondition}=

Set Variable If

'${n}'<='${maxIndexLogicalOpeator}'

${tempCondition} @{operatorList}[${n}]

${tempCondition}









[Return]

${whereClauseCondition}

Create mkv.jinit File Required For Running DB Deploy Tool

[Arguments]

${PathWhereFileIsToBeCreated}

${DBType}

${DBHost}

${DBPort}

${DBName}

${DBUser}

${DBPwd}

${InputScriptDir}

${OutputDir}=${EMPTY}





#### VARIABLE CONTAINING CONTENTS OF MKV.JINIT FILE










@{ContentsOfFile}=

Create List













...

\#DBMS type: sybase, oracle or mssql\r













...

mkv.db.type = ${DBType.lower()}\r\r













...

\#machine DBMS is running on, host name or ip address\r













...

mkv.db.host = ${DBHost}\r\r













...

\#port of the DBMS\r













...

mkv.db.port = ${DBPort}\r\r













...

\#name of the DB\r













...

mkv.db.name = ${DBName}\r\r













...

\#user id to access to the DB\r













...

mkv.db.user = ${DBUser}\r\r













...

\#password to access to the DB\r













...

mkv.db.password = ${DBPwd}\r\r













...

\#path of folder where the upgrade scripts are available\r













...

mkv.db.input_script_dir = ${InputScriptDir}\r\r













...

\#path of the folder where the upgrade and rollback scripts will be generated by dbdeploy\r













...

mkv.db.output_dir = ${OutputDir}\r\r
